# Data Model: Local VSIX Packaging & Install Scripts

**Feature**: 002-local-vsix-install  
**Phase**: 1 — Design  
**Date**: 2026-02-28

## Overview

This feature introduces no persistent data entities, database schemas, or application state.
It is pure build tooling. This document records the relevant artefact types, their
locations, and the constraints that govern them.

---

## Artefact: `.vsix` Package File

| Attribute | Value |
|-----------|-------|
| Location | `releases/` at project root |
| Filename pattern | `logexplorer-{version}.vsix` where `{version}` matches `package.json#version` |
| Generated by | `vsce package --out releases/ --allow-missing-repository` |
| Consumed by | `code --install-extension releases/logexplorer-{version}.vsix` |
| Lifecycle | Created on each `npm run package:local`; overwrites previous file of the same name |
| Retention | Not version-controlled (`releases/` in `.gitignore`); developer manages manually |
| Contents | Compiled extension bundle (`dist/`), `package.json`, `README.md`, `LICENSE`, icon |

**Validation rules**:
- File MUST exist before `install:local` is invoked; otherwise the install script exits with code 1.
- Filename MUST match `*.vsix`; any other file in `releases/` is ignored.

---

## Artefact: `releases/` Directory

| Attribute | Value |
|-----------|-------|
| Location | Project root |
| Created by | `scripts/package-local.mjs` via `fs.mkdirSync('releases', {recursive: true})` |
| Excluded from Git | Yes — `releases/` entry in `.gitignore` |
| Excluded from VSIX | Yes — `releases/**` entry in `.vscodeignore` |
| Contents | One or more `.vsix` files (typically one, overwritten on each build) |

---

## Script Files

These are the new source files introduced by this feature:

| File | Type | Purpose |
|------|------|---------|
| `scripts/package-local.mjs` | Node.js ESM script | Creates `releases/` dir, invokes `vsce package` |
| `scripts/install-local.mjs` | Node.js ESM script | Finds `.vsix` in `releases/`, invokes `code --install-extension` |

**Script constraints**:
- MUST run under Node.js 18+ (`child_process.execSync`, `fs` are standard).
- MUST be cross-platform (no OS-specific shell commands).
- MUST exit with code 0 on success and non-zero on failure.
- MUST print a human-readable error message to stderr/stdout on failure.
- MUST NOT make network calls or transmit artefacts externally.
